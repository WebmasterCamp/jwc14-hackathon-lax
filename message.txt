# Web Application Prototype (High-Fidelity MVP): readdiva.in.th

**Role:** Expert Senior Full-Stack Developer, Frontend Engineer, UI/UX Designer, and Product Designer.

---

## 1. Main Concept
A Gamification platform to encourage users to read more books by turning "reading time" into "power to conquer territories." Users will accumulate study time from various Sessions to increase their Influence score and conquer areas on the map. The person who goes to read in a specific Co-working space will accumulate time in that area, and if they are the user with the highest reading time, they will conquer that area. On the map, it will display that person's Icon, and the area will change color according to the color of the Top 1 user. The system must make users feel like they are building their own empire of knowledge through reading.

## 4. Theme and Design System

* **Color Palette:**
  * **Primary:** Deep Black, Dark Gray, Midnight Purple

  * **Accent:** Orange (#FF8A00)
  * Use orange for CTA Buttons, Charts, Achievements, Notifications.
* **Background:** Gradient (Black → Midnight Purple → Indigo) with a subtle Glow Effect.
* **UI Style:** Glassmorphism, Rounded Corners, Floating Cards, Soft Shadows, Smooth Animation, Framer Motion.
* **User Flow:** Enter the home page and see a Map to select the area to study (which are real-world locations) -> Select Session time and start the reading Session -> Accumulate time, can select Focus mode -> Gain Influence -> Conquer territory -> Earn Achievement -> Compete with friends on the Leaderboard.
* Users must understand this concept within the first 5 seconds.
* For each Menu, primarily use a Bottom Menu Bar with the following menus: `[Home, Leaderboard, History, Achievements, Avatar]`.

## 14. Map Pins Support
Design and implement a 2D Interactive Map for the `readdiva.in.th` prototype using real-world geography of Bangkok, Thailand. The map must use React Leaflet or Mapbox GL JS and strictly follow the 'Election Map' gamification concept:
* **Base Map Customization (Default Area):** Use a highly customized, minimalist map style. Strip away all default colors, terrain details, and minor roads. The landmass of Bangkok should be clean white or very light gray. Major roads or district borders can be extremely faint just to provide basic geographical context.
* **Real-world Coordinates & Unclaimed Zones:** Use actual Latitude/Longitude data for real cafes and co-working spaces in Bangkok (e.g., Siam, Samyan areas). Unclaimed locations are displayed only as modern 'Map Pins' on the white background, with no background highlight color, indicating they are open for conquest.
* **Conquered Zones (GeoJSON Polygons):** For locations where a user is Top 1, display a 'Map Pin containing the conqueror's icon'. Surround the pin with a dynamically generated GeoJSON polygon (representing the territory's radius or district) and fill it with the Top 1 user's signature color. This must look like an electoral district conquered by a specific party, contrasting beautifully against the white base map.

## 33. Location Details Popup & Local Room View Feature
* **Location Modal:** When a user clicks on a specific territory or Map Pin, a Popup/Modal will appear containing the following details:
  * **Co-Working Space Name:** Clearly display the name and type of the location.
  * **Local Leaderboard:** As the user scrolls down, display the top-ranking users who have accumulated the most study time specifically in this location.
  * **Sync Location Button:** A Call-to-Action (CTA) button allowing users to "Sync" or "Check-in" to confirm their virtual or physical presence in this space.
* **Session Setup:** After clicking the Sync button, the standard Session Timer setup will appear, allowing the user to select their desired study duration (e.g., 15, 30, 45, 60 minutes).
* **Local Mini-Map (Active Session View):**
  * Once the session starts, instead of just showing a standard timer, the UI will transition into a "Mini-Map" or an isometric room view specific to that co-working space.
  * This view will display the Avatars/Icons of other users who are currently studying in the same location at that moment (using Mock Data).
  * **Objective:** To boost motivation by creating a sense of community and shared focus, letting users visually see that they are not studying alone.
